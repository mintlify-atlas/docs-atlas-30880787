---
title: "Run Zero locally: clone, configure, and launch in minutes"
description: "Set up Zero locally using pnpm and Docker. Covers prerequisites, environment configuration, Google OAuth, database setup, and starting the dev server."
---

Zero is a pnpm monorepo with a React Router v7 frontend (built with Vite) and a Node.js backend (running on Cloudflare Workers). This guide walks you through every step needed to get a fully working local instance running at `http://localhost:3000`—from cloning the repository to signing in for the first time.

## Prerequisites

Make sure the following tools are installed before you begin:

| Tool | Required version |
|------|-----------------|
| [Node.js](https://nodejs.org/en/download) | v18 or higher |
| [pnpm](https://pnpm.io) | v10 or higher |
| [Docker](https://docs.docker.com/engine/install/) | v20 or higher |

<Tip>
  Run `pnpm go` after completing the environment setup to start the database and all dev servers in one command.
</Tip>

## Setup

<Steps>
  <Step title="Clone the repository and install dependencies">
    Clone the Zero repository and install all workspace packages with pnpm.

    ```bash
    git clone https://github.com/Mail-0/Zero.git
    cd Zero
    pnpm install
    ```

    <Note>
      `pnpm install` automatically runs `pnpm nizzy sync` via the `postinstall` hook, which generates typed environment variable helpers from your `.env` file. You will run it again manually after configuring your environment in the next step.
    </Note>
  </Step>

  <Step title="Configure environment variables">
    Zero ships with a CLI tool called `nizzy` that scaffolds your `.env` file from the provided example. Run it from the repository root:

    ```bash
    pnpm nizzy env
    ```

    This copies `.env.example` to `.env`. Open the newly created `.env` file and fill in the required values:

    ```bash
    # Generate a secure auth secret
    openssl rand -hex 32
    ```

    ```env
    # Auth
    BETTER_AUTH_SECRET=<output from openssl above>
    BETTER_AUTH_URL=http://localhost:3000

    # Google OAuth — required for Gmail integration
    GOOGLE_CLIENT_ID=your_client_id
    GOOGLE_CLIENT_SECRET=your_client_secret

    # Database — default matches the Docker Compose setup below
    DATABASE_URL="postgresql://postgres:postgres@localhost:5432/zerodotemail"

    # Redis — default matches the Docker Compose setup below
    REDIS_URL="http://localhost:8079"
    REDIS_TOKEN="upstash-local-token"

    # Encryption service
    AUTUMN_SECRET_KEY=your_autumn_secret
    ```

    After editing `.env`, sync the types:

    ```bash
    pnpm nizzy sync
    ```

    <Warning>
      The Google OAuth redirect URI must be set to `http://localhost:8787/api/auth/callback/google` in your Google Cloud Console project, or sign-in will fail. See the [Google OAuth setup guide](https://console.cloud.google.com) for details.
    </Warning>
  </Step>

  <Step title="Set up Google OAuth">
    Zero requires Google OAuth to connect Gmail accounts.

    1. Open the [Google Cloud Console](https://console.cloud.google.com) and create a new project.
    2. Enable the [Gmail API](https://console.cloud.google.com/apis/library/gmail.googleapis.com) and the [People API](https://console.cloud.google.com/apis/library/people.googleapis.com).
    3. Go to **APIs & Services → Credentials** and create an **OAuth 2.0 Client ID** of type **Web application**.
    4. Under **Authorized redirect URIs**, add:
       ```
       http://localhost:8787/api/auth/callback/google
       ```
    5. Copy the **Client ID** and **Client Secret** into your `.env` file.
    6. Go to [**Audience**](https://console.cloud.google.com/auth/audience) and add your email address as a test user.
  </Step>

  <Step title="Start the database">
    Zero uses PostgreSQL for persistent storage and Valkey (a Redis-compatible store) for caching. The `docker-compose.db.yaml` file starts both services along with an Upstash-compatible HTTP proxy for the Redis client.

    ```bash
    pnpm docker:db:up
    ```

    This starts three containers:

    | Container | Image | Port |
    |-----------|-------|------|
    | `zerodotemail-db` | `postgres:17` | `5432` |
    | `zerodotemail-redis` | `bitnami/valkey:8.0` | `6379` |
    | `zerodotemail-upstash-proxy` | `hiett/serverless-redis-http` | `8079` |

    The default database credentials (`postgres` / `postgres`) match the `DATABASE_URL` in `.env.example`.
  </Step>

  <Step title="Push the database schema">
    Apply the Drizzle schema to your local database:

    ```bash
    pnpm db:push
    ```

    This command introspects the schema defined in `packages/db` and creates all required tables without generating migration files. Run it again any time you pull schema changes from upstream.

    <Note>
      To open a visual browser for your database, run `pnpm db:studio`. When you run `pnpm dev` the studio starts automatically alongside the app.
    </Note>
  </Step>

  <Step title="Start the development servers">
    Start all apps and packages in watch mode using Turborepo:

    ```bash
    pnpm dev
    ```

    Turborepo starts the Next.js frontend and the backend server in parallel. Wait for both to report that they are ready, then open your browser.

    ```
    http://localhost:3000
    ```

    Sign in with the Google account you added as a test user in step 3. Zero will sync your inbox and you can start exploring the app.
  </Step>
</Steps>

## Common development commands

| Command | Description |
|---------|-------------|
| `pnpm go` | Start database and all dev servers together |
| `pnpm docker:db:down` | Stop and remove database containers |
| `pnpm docker:db:clean` | Stop containers and delete volumes |
| `pnpm db:generate` | Generate Drizzle migration files after schema changes |
| `pnpm db:migrate` | Apply pending migrations |
| `pnpm db:studio` | Open Drizzle Studio in the browser |
| `pnpm nizzy sync` | Regenerate typed env helpers after `.env` changes |
